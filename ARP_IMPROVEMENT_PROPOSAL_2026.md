## 1. Simulated / Heuristic / Validated 명시적 구분

### 문제점
현재 구조에서 실제 데이터와 시뮬레이션 결과가 섞여 보일 수 있음

### 제안: Evidence Level 4단계

```python
class EvidenceLevel(Enum):
    REAL_DATA = "real"           # 실제 실험/임상 데이터
    VALIDATED = "validated"       # 검증된 모델 출력
    HEURISTIC = "heuristic"       # 휴리스틱/규칙 기반
    SIMULATED = "simulated"       # 시뮬레이션 결과
```

### 출력 예시

```json
{
  "targets": [
    {
      "name": "FSP1",
      "evidence_level": "validated",
      "confidence_source": "Boltz-2 binding prediction",
      "validated_by": ["Nature 2025", "PNAS 2025"]
    },
    {
      "name": "SLC7A11",
      "evidence_level": "real",
      "confidence_source": "TCGA-LUAD bioinformatics",
      "source": "Larry report, TCGA 4.7x overexpression"
    }
  ]
}
```

### 문서 분리

```
ARP_Output/
├── real_data/          # 실제 실험 데이터
├── validated/          # 검증된 예측
├── heuristic/          # 규칙 기반
├── simulated/          # 시뮬레이션
└── metadata/           # 출처 메타데이터
```

## 3. CI / Smoke Tests (GitHub Actions)

```yaml
name: ARP Pipeline Tests

on:
  push:
    branches: [main, yars2-sirna-pf14-2026]

    branches: [main]

  smoke-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      
      - name: Install dependencies
        run: pip install -r requirements.txt || true
      
      - name: Import check
        run: python3 -c "from arp_orchestrator import ARPOrchestrator"
      
      - name: CLI help check
        run: python3 arp_orchestrator.py --help | grep -q "playbook"
      
      - name: Boltz2Client import
        run: python3 -c "from boltz2_client import Boltz2Client"
      
      - name: Run unit tests

        "retry": 1,
        "timeout_seconds": 600,
        "fallback": "template_based",
        "skip_on_failure": True,
        "partial_result": True,
        "alert_on_failure": True
    },
    "admet_prediction": {
        "retry": 2,
        "timeout_seconds": 180,
        "fallback": "historical_data",
        "skip_on_failure": True,
        "partial_result": True,
        "alert_on_failure": False
    }
}
```

### Failure Recovery Plan

```
Stage fails
    ↓
Check retry count
    ↓
If retry < max: retry
    ↓
If retry >= max: apply fallback policy
    ↓
If fallback exists: continue with fallback
    ↓
If no fallback: mark stage as skipped, log error, continue
    ↓
Always: save partial_result + alert
```

## 7. Branch/Version 정리 가이드

### 제안: Branch Strategy

```
main                    ← stable, fully tested, 문서화 완료
├── yars2-sirna-pf14-2026  ← current working branch (이슈 처리 중)
├── feature/leader-agent   ← experimental feature
└── archive/              ← 오래된 branch들 (read-only)
```

### Branch Naming Convention

| Prefix | Purpose | Example |
|--------|---------|---------|
| `main` | 안정版 | `main` |
| `yars2-*` | 프로젝트별 | `yars2-sirna-pf14-2026` |
| `feature/` | 새 기능 | `feature/leader-agent` |
| `hotfix/` | 긴급 수정 | `hotfix/critical-bug` |
| `archive/` | 보관 | `archive/old-version` |
